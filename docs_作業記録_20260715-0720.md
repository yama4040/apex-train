## 付録: 現在の報酬NN設計の詳細（2026-07-20時点、train_reward_network2.py）

### A. 全体構成 — 2段階ハードルモデル

1つの回帰NNに「reward=0.0か否かの判定」と「非0.0なら何点か」を同時にやらせると、
0.0の山（データの約4割）との補間で本来0.0の状況に0.1〜0.3を漏らす（DQNがその漏れを搾取する）ため、
仕事を2つのモデルに分離している。

```
入力（46次元・共通のStandardScalerで正規化）
   ├─▶ ゲート分類器: 「この操作はreward=0.0か？」→ P(0.0) をsigmoidで出力
   └─▶ 回帰器:       「0.0でないなら何点か？」→ 連続値（非0.0の行のみで学習）

推論（direct_reward_predictor2.py）:
   P(0.0) ≥ 0.5 → reward = 0.0
   それ以外     → reward = clip(回帰値, 0.1, 1.0) を0.1刻みに丸め
（環境側でさらにゼロ中心化＋time_stepスケール: (reward−0.5)×(Δt/1.0)、goal時はイベント報酬として×1.0満額）
```

- ゲートは全データで学習（正解ラベル: reward ≤ 0.05 → 「0.0クラス」。ZERO_THRESHOLD=0.05）
- 回帰器は reward > 0.05 の行のみで学習（0.1〜1.0の値域に専念させ、0.0の山に引っ張られないようにする）
- 分離の効果実測: ラベル0.0への報酬リーク（予測>0.25）15.7% → 1.4%（導入時点）

### B. 入力特徴量（46次元）

- 数値35: ノッチ別保持時間（hold_coast/accel/decel）、prev_notch_duration、速度・距離・時間・遅延・勾配の生値、
  マージン系（margin_speed / margin_signal_speed / margin_stop_dist）、必要速度系（required_speed / speed_margin_to_required）、
  ノコギリスコア（hunting_score、閾値7秒）、相対速度、前方制限・勾配のパース値、先行・後続列車の存在/距離/速度
- **クリップ済み特徴3つ（このNNの要）**:
  - margin_stop_dist_clip = clip(Δ, −30, +30)m … ブレーキ開始判断レンジ
  - dist_to_station_clip = clip(残距離, −20, +20)m … 停止誤差レンジ
  - dist_to_station_clip3 = clip(残距離, −3, +3)m … 1m段差レンジ
  - 理由: 生のΔ・距離はσ≈775〜793mで、±5mの境界が0.0064σ・±1mが0.0013σとなり標準化後にNNから見えない。
    クリップ後はσ≈15m／10m／0.9mになり、境界がそれぞれ0.3σ／0.1σ／1.1σの「見える」スケールになる
- one-hot 11: 走行フェーズ5（加速/巡航/制限接近/減速/停車完了）＋現在ノッチ3＋直前ノッチ3
- 正規化: StandardScaler（学習データでfit、ゲート・回帰器で共有、direct_reward_scaler2.pklとして保存）
- 世代互換: 推論側はスケーラーの n_features_in_ で 43/45/46次元を自動判別し、旧モデルでも動作する

### C. モデル構造（ゲート・回帰器とも同一トランク）

```
Input(46)
→ Dense(128, L2=1e-4) → BatchNormalization → ReLU → Dropout(0.3)
→ Dense(64,  L2=1e-4) → BatchNormalization → ReLU → Dropout(0.2)
→ Dense(32,  L2=1e-4) → BatchNormalization → ReLU → Dropout(0.1)
→ 出力層:  回帰器 Dense(1, linear)  ／  ゲート Dense(1, sigmoid)
```

- BatchNorm＋L2＋段階的Dropoutは、出力が0〜1レンジから大きく外れる不安定学習を防ぐための構成
  （回帰の出力層はあえてsigmoidにせずlinearのまま。値域の収束は学習の安定化側で促す設計）

### D. 損失の計算

損失は「行ごとの誤差 × サンプル重み」の平均。

- **回帰器**: Huber損失（delta=1.0）。二乗誤差より外れ値（LLM評価のゆらぎ・ラベルノイズ）に頑健
- **ゲート**: binary_crossentropy
- **サンプル重み（回帰器）** = ①ビン重み × ②フェーズ重み、最後に平均1.0へ再正規化:
  - ① ビン重み: ラベルを0.1刻みの11ビンに分け、重み ∝ 1/√(ビン件数)。
    単純な逆頻度だと0.0ビン（数千件）と0.7ビン（数十件）の重み比が150倍超になり、
    多数派領域の誤差が事実上放置される（過去に報酬リークの主因になった）ため√で緩和し比を10倍強に抑える
  - ② フェーズ重み: 停車完了フェーズの行に STOP_PHASE_WEIGHT=3.0。
    停車完了は全体の約9%しかなく回帰の平滑化で1m段差の頂点が潰れる（教師平均0.957に対し出力0.9止まり）ため、
    「停車完了での0.05のずれ＝他領域の0.15のずれ」と等価の罰金にして優先的に合わせさせる
  - 平均1.0への再正規化は、重み導入が実効学習率を変えないようにするため
- **サンプル重み（ゲート）**: フェーズ重み（停車完了×3.0）のみ。良い停車への誤0.0判定（外れ値）の抑制が目的
- 補助指標: MAEと custom_accuracy（|予測−ラベル| ≤ 0.15 を正解とみなす率）。学習には使わずモニタ用

### E. 学習率の調整と学習の停止

- オプティマイザ: Adam（初期学習率 1e-3）＋ **勾配クリッピング clipnorm=1.0**（外れ値入力による重みの急発散防止）
- **ReduceLROnPlateau**: val_lossが10エポック改善しなければ学習率を×0.5（下限 1e-6）。
  局所解での停滞・発散を防ぐ段階的減衰
- **EarlyStopping**: val_lossが30エポック改善しなければ停止し、**最良時点の重みへ復元**（restore_best_weights=True）
- 最大500エポック・バッチサイズ64（実際は早期終了で数十〜200エポック程度で収束）
- データ分割: 80/20、乱数シード42固定、**0.1刻みラベルビンで層化**（少数ビンが検証側に偏らないように）

### F. 学習後の自動評価（mainが出力）

- 合成モデル（ゲート＋回帰を推論と同じロジックで合成）でのテストMAE
- ラベル0.0のうち予測>0.25の割合（報酬リーク率）と平均予測
- 非0.0ラベルのMAE
- これとは別に、再学習のたびに手動プローブ検収（ブレーキ判断マップ・停車完了マップ・搾取チェック等。引継ぎ資料§6参照）を実施する運用
